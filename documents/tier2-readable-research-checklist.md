### Confirmed mapping rules from repo docs/code

1. Zillion index mapping

- For 10^p, power index is based on (p - 3) / 3.
- For million/billion style naming, code decrements by 1 for zillions:
  - N=2 gives million,
  - N=3 gives billion,
  - and so on.

1. Indefinite extension rule (critical)

- Quote in repo README (from Conway/Wechsler convention):
  - XilliYilliZillion means the (1000000X + 1000Y + Z)th zillion.
  - nillion is used as a zeroth placeholder when needed.
  - Example: million-and-third zillion => millinillitrillion.

1. Prefix construction is Conway-Wechsler grammatical

- Ones/tens/hundreds segments are built with inflection markers and join rules.
- It includes letter insertion behavior (for example matching marker rules) and trailing-vowel handling before adding illion.

1. Useful practical implication for this project

- Readable names after 1e3006 should not be generated by a simple tier2-prefix + tier1-name shortcut.
- They should be composed from grouped 3-digit zillion indices with nilli placeholders where needed.
